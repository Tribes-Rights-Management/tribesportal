## Purpose

Define the minimum exports required to demonstrate authority governance before auditors ever log in.

---

## Core Principle

> **Auditors don't need access if exports are clean.**  
> **Access is a later convenience, not a prerequisite.**

---

## A. Authority History Export (Required)

### Purpose

Answers the #1 question auditors ask:

> **"Who had the right to do what, at this time?"**

### Contents

| Field | Description |
|-------|-------------|
| Event timeline | Chronological sequence |
| Actor | Who performed the action |
| Target user | Who was affected |
| Scope | Platform or organization |
| Action | Granted / Removed / Approved / Declined |
| Timestamp | UTC, server-generated |
| Approval reference | If approval was required |
| Reason | If provided |

### Format

- **Primary**: PDF
- **Layout**: Human-readable, grouped chronologically
- **Header**: "System-generated authority record"

### Export Structure

```
TRIBES AUTHORITY HISTORY EXPORT
Generated: 2026-01-17T14:30:00Z
Generated by: Sarah Lee (sarah@company.com)
Scope: Platform
Period: 2025-01-01 to 2026-01-17

───────────────────────────────────────────────────

Jan 14, 2026 • 10:32 AM UTC
Adam Carpenter granted Organization Administrator to Jordan Smith
Scope: Tribes Licensing
Approved by: Sarah Lee (Jan 14, 2026 • 2:15 PM UTC)

Jan 10, 2026 • 9:00 AM UTC
System granted Platform User to Jordan Smith
Scope: Platform
Reason: Initial account creation

...
```

## B. Current Authority Snapshot (Required)

### Purpose

Answers the immediate question:

> **"What is true now?"**

Pairs with history export to show both present state and how we arrived here.

### Contents

| Field | Description |
|-------|-------------|
| User | Name and email |
| Platform role | Current platform authority |
| Organization memberships | List of orgs and roles |
| Context access | Publishing, Licensing, etc. |
| Status | Active, Suspended, etc. |

### Format

- **Primary**: PDF
- **Layout**: User-by-user breakdown
- **Header**: "Current authority snapshot"

### Export Structure

```
TRIBES CURRENT AUTHORITY SNAPSHOT
Generated: 2026-01-17T14:30:00Z
Generated by: Sarah Lee (sarah@company.com)
Scope: Platform

───────────────────────────────────────────────────

JORDAN SMITH
jordan@company.com

Platform Authority
  Role: Platform User

Organization Memberships
  
  Tribes Licensing

    Status: Active
    Since: Dec 1, 2025

───────────────────────────────────────────────────

SARAH LEE
sarah@company.com

Platform Authority
  Role: Platform Executive

...
```

## C. Approval Decisions Export (Required Once Approvals Exist)

### Purpose

Proves governance discipline, not just access control.

### Contents

| Field | Description |
|-------|-------------|
| Proposed change | What was requested |
| Proposer | Who requested |
| Approver | Who decided |
| Decision | Approved / Declined |
| Timestamp | When decided |
| Scope | Platform or organization |
| Reason | If provided |

### Format

- **Primary**: PDF
- **Layout**: Decision-by-decision breakdown
- **Header**: "Approval decisions record"

### Export Structure

```
TRIBES APPROVAL DECISIONS RECORD
Generated: 2026-01-17T14:30:00Z
Generated by: Sarah Lee (sarah@company.com)
Scope: Platform
Period: 2025-01-01 to 2026-01-17

───────────────────────────────────────────────────

APPROVED • Jan 14, 2026 • 2:15 PM UTC

Proposed change:
  Grant Organization Administrator to Jordan Smith
  Scope: Tribes Licensing

Proposed by: Adam Carpenter
Approved by: Sarah Lee
Reason: "Promoted to lead publishing operations"

───────────────────────────────────────────────────

DECLINED • Jan 8, 2026 • 11:00 AM UTC

Proposed change:
  Grant Platform Administrator to Mike Johnson
  Scope: Platform

Proposed by: Adam Carpenter
Declined by: Sarah Lee
Reason: "Requires executive committee approval per policy"

...
```

## D. What NOT to Export Yet

These increase risk without adding trust:

| Exclude | Why |
|---------|-----|
| Raw permission flags | Implementation detail |
| Internal IDs | Not human-readable |
| System notes | Internal only |
| Cross-org combined datasets | Scope violation |
| Database keys | Security risk |

## F. Export Logging

Every export is logged as an immutable event:

```typescript
{
  event_type: 'authority_export_generated',
  actor_id: string,
  actor_email: string,
  export_type: 'history' | 'snapshot' | 'approvals',
  scope: 'platform' | 'organization',
  organization_id?: string,
  date_range_start?: string,
  date_range_end?: string,
  record_count: number,
  reference_id: string,
  created_at: string
}
```
